# 框架变更记录 · 画像自进化框架

> 每条标注**是否需迁移动作**：🟢 无需迁移（纯规则，覆盖即可）／⚠️ 需迁移（结构变化，人工核对）／🧩 增量补（画像内部约定，增补不重写）。
> 升级改造旧项目时，从旧项目当前版本逐条向下执行到最新版。流程见 `使用指南.md §二`。

---

## v1.0.0 — 2026-06-15（首版）

首次从 planX 实例抽象成可复用框架。确立：

- 🟢 **信息四层模型**：贴源／讨论总结／人物画像／产出文件，同一信息只在一层权威存放。
- 🟢 **目录与命名规范**：`第NN次_YYYY-MM-DD_主题`，按次归档与连续编号。
- 🟢 **分层加载策略**：必读（规则+画像）／选读（最近1次总结）／按需读（grep 历史），固定开销与讨论次数无关。
- 🟢 **强制三件套**：每次讨论后 贴源归档 + 讨论总结 + 画像刷新。
- ⚠️ **画像三件套结构**：正文（结论）／更新日志（索引层，一行一条）／完整存档（兜底）。
- 🧩 **自进化机制**：累积刷新、最近更新置顶、重要级与状态标记（🔴/✅/⚠️/🎯/moot）、阅读前提块+决策底线矩阵、编号小节累积、诤臣留痕。
- 🟢 **贴源图片转文字即删原图**规则。

模板产物：`模板/CLAUDE.md`、`模板/人物画像_模板.md`。
