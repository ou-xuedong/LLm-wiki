# <项目名> —— 项目规则

> 本项目用于沉淀主人与 **<对象>** 的每一次深度讨论：归档原始素材、提炼讨论结论、持续刷新人物画像。
> 本文件是落盘规则，每次在本目录下工作都必须遵守。

## 目录结构

**信息四层模型（理解全框架的总纲）**：同一信息只在一层"权威"存放，不跨层重复——这是防膨胀/防回音壁的根本设计。
① 事实层＝贴源层　② 过程层＝讨论总结　③ 状态层＝人物画像（正文＝结论 / 更新日志＝索引 / 完整存档＝兜底）　④ 物料层＝产出文件。

```
<项目名>/
├── CLAUDE.md                          # 项目规则（本文件）
├── 贴源层/                             # ① 事实层：原始素材，只记录不分析
│   ├── 第NN次_日期_主题/               #    按次归档（图片转文字、转完删图）
│   └── 背景补充_日期_主题/             #    跨次背景素材
├── 每次的讨论总结/                      # ② 过程层：一次讨论一文件，展开复盘
│   └── 第NN次_日期_主题.md
├── 人物画像/                           # ③ 状态层：一人一画像，累积刷新＝最新结论
│   ├── <对象>.md                      #    画像正文（结论态；每次会话必读）
│   └── <对象>_更新日志.md              #    索引层（一行一条：主题→画像节→总结；grep 首选入口）
└── 产出文件/                           # ④ 物料层：拿去就用的话术/提纲/方案（非长期记忆、可清理）
    └── backUp/                        #    历史版本归档
```

## 各目录的用途与落盘规则

### 1. 贴源层
- 放**主人每次给的背景内容**：原始对话（截图 + 文字还原）、相关报告/文件等。
- **按次分子目录**：`第NN次_YYYY-MM-DD_主题/`，该次所有素材放进去。
- 原则：只记录事实，**不做分析、不加评论**。
- **图片必须转文字，转完即删原件**：遇截图/图片第一时间读图做**完整文字还原**存 `.md`；校对无误后删除图片原件——省空间省 token。
- ⚠️ 删图不可逆，删前务必确认文字还原完整（含发言人、顺序、关键非文字信息如表情/时间）。

### 2. 每次的讨论总结
- 一次讨论 = 一个文件：`第NN次_YYYY-MM-DD_主题.md`。
- 内容应包含：事件经过、核心分歧、谁对谁错、关键洞察、（如涉及沟通/关系）可复用的策略与教训、待跟进项。
- 顶部注明对应的贴源子目录。

### 3. 人物画像
- 每个对象一个文件：`<对象>.md`。
- **累积刷新**：不按次拆分，始终维护一份最新画像。
- **顶部维护"最近更新"摘要**：每次讨论追加一条、**最新在前**、高度压缩（日期/次号/本次最重洞察/改了哪节）。
- **更新日志独立为索引层**：每次更新往 `人物画像/<对象>_更新日志.md` 追加**一行索引**（格式：`日期｜次号/场景｜主题一句｜画像锚点｜来源(总结/贴源)`），**不复述内容**——展开看「讨论总结」、最新结论看画像正文，**三处不重复**（防膨胀/防回音壁的核心）。
- 应覆盖：身份、业务/专业背景、世界观价值观、沟通风格、性格、与主人的关系、相处注意事项（雷区/加分）。
- **重要级与状态标记**（统一约定，便于扫读）：🔴 诤臣/警示、🔴🔴 钥匙级、✅ 推荐、⚠️ 注意/存疑、🎯 关键判断、【已坐实】【已发生】、**moot/已过时仅备查**（历史预案作废保留，防当现行）。

### 4. 产出文件
- 放**主人在与对象交往/沟通过程中要实际用到的中间材料**：提纲、话术、方案、文档草稿、活动策划等。
- 分工：贴源层＝原始事实，讨论总结＝复盘洞察，画像＝长期记忆，**产出文件＝拿去就能用的"作战物料"**。
- 命名：`<用途简述>_<可选日期/对象>.md`，文件顶部一行注明用途与适用场景。
- 是工作产物、非长期记忆：**不强制读取**，过期/用完可清理。

## 强制规则（每次讨论必做）

1. **每次讨论后，必须更新相关对象的画像**（`人物画像/<对象>.md`），并在其更新日志追加记录。硬性步骤，不可跳过。
2. 每次讨论必须三件套齐全：① 贴源归档 ② 讨论总结 ③ 画像刷新。
3. **命名规范**：次号统一 `第NN次`（两位，01、02…），日期用绝对日期 `YYYY-MM-DD`，主题用简短中文。
4. **次号递增**：开新讨论前，先看现有最大次号 +1。同一对象的讨论在本项目内连续编号。
5. **贴源图片一律转文字，转完即删图片原件**——只保留文字版，省空间省 token；删前确认还原完整。

## 会话加载策略（防 token 爆炸）

讨论会一次次累积，**严禁每次会话全量读取**。按三层加载：

- 🟢 **必读（定长）**：`CLAUDE.md` + 对应 `人物画像/<对象>.md`。画像累积刷新，历次精华已沉淀其中——读画像≈读了全部历史精华。两份长度基本恒定。
- 🟡 **选读**：`每次的讨论总结/` 里**最近 1 次**总结，用于接上次的线。
- ⚪ **按需读**：更早的讨论总结、贴源层原文——**仅在需核对原话/回溯细节时**读对应那一次，用完即止。

固定开销 = 规则 + 画像 + 最近 1 次总结，**与讨论总次数无关**。历史细节靠按需检索，不靠全量背诵。

### 何时 grep（按需检索触发规则）
- 主人问**原话/哪次/具体细节/时间** → grep `贴源层/**` 或 `每次的讨论总结/**`，回原始记录核对。
- 引用**带日期或次号的事件** → 先 grep 对应对象的 `_更新日志.md`（索引层）拿到"画像哪节 + 哪个总结"，再精准展开那一条。
- 画像结论与**主人当下所说有出入** → grep 总结/贴源核对依据再判（防画像过时或被误读）。
- 默认：能 grep 定位到的就不通读整文件；索引层 `_更新日志.md` 是 grep 的首选小目标。
