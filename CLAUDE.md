# AILab 项目规则

> 本文件为 AILab 目录的项目级规则，与 `~/CLAUDE.md`（人格 + 工作准则）叠加生效。
> 优先级：当下指示 > 本文件 > `~/CLAUDE.md` > 系统默认。

---

## 规则一：生成 HTML 报告前，先加载报告规则（强制）

凡是要生成 **HTML 格式的报告**，动手前**必须先 Read 并遵守**：

```
.claude/skills/html-report/SKILL.md
```

该文件含两条强制规则：**术语悬停注释**（小白看得懂）+ **护眼视觉**（白底、护眼字号排版）。

> 细则按需加载，不在本文件内联，节省每次常驻 token。

---

## 规则二：涉及主人个人信息时，先读自我画像（强制）

凡涉及**主人本人**的事——写简历、谋划职业/创业方向、个人决策咨询、自我相关分析等，动手前**必须先 Read**：

```
ProjectT/自我画像/自画像.md       # 主人是谁：背景、性格、行为模式、当前处境与目标
ProjectT/自我画像/教训与原则.md    # 经历提炼的教训与可引用原则（P-NN）
```

两份是累积刷新的长期记忆，**先读再给方案**，确保背景准确、不臆断。

---

## 规则三：研究开源项目吸收模块前，先调用借鉴 skill（强制）

凡涉及**研究 GitHub/开源 AI 项目、借鉴或吸收其模块**进主人自有项目（记忆/规划/反思/工具调用/自进化等），触发词如"研究下XX项目""XX能借鉴什么""想把XX集成进来"，动手前**必须先调用 skill**：

```
borrow-github-module
```

该 skill 已编码主人定位（模块借鉴整合派/器官移植医生）+ 三阶段方法论（A 理解 → B 集成 → C 多源冲突），并强制先用 `AskUserQuestion` 补全需求，不凭空假设。
