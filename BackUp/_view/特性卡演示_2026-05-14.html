#FC-001 P0 🧪 testing

MACD 底背离识别（日线，单标的）

Epic: 技术面分析体系 · 估时 ~6h · 实际 4h32m · 创建 2026-05-13

需求段 来源：主人交易风格 / MEMORY 记录

🎭 角色 / 场景

主人在每日盘后扫描自选 ETF 池，希望系统自动标出当前在日线级别出现 MACD 底背离信号的标的，作为潜在加仓候选清单。

✅ 验收标准（≤5 条）
  1. 输入：标的代码 + 日线 K 线数据（含至少 60 根）
  2. 输出：True/False + 置信度（0-1）+ 命中的两个低点日期
  3. 识别规则：价格创新低 + MACD 柱状图低点抬高 + DIF 抬高
  4. 已知样例可全部正确识别（提供 5 条历史正样本 + 3 条负样本）
  5. 单标的判定 ≤ 200ms

产品段 特性描述 + 数据流

📝 特性描述

一个纯函数 detect_macd_bottom_divergence(kline_df) → DivergenceResult，无副作用，单一职责，供上层 ETF 扫描卡调用。

🔁 数据流
K 线 DataFrame
  ↓ 计算 MACD（DIF / DEA / 柱状图）
  ↓ 找最近两个相邻价格低点
  ↓ 比较低点价格 vs MACD 值
  ↓ 输出判定 + 置信度

技术段 方案 + 接口契约

🏗 实现方案

  • 位置：src/signals/macd_divergence.py
