# Agent 驱动研发框架

> AILab 体系的研发执行框架。被业务项目读取，让"需求 → PRD → 研发 → 测试 → 上线"形成可追踪、可迭代、人在环的自动化流水线。

- **版本**：v0.1 草案 · 2026-05-14
- **阶段**：方案讨论已收口，待打磨
- **下一步**：下次会话继续——4A 模板细化、Agent 协议、需求看板 HTML 原型

## 入口

- 🖼️ [框架全景.html](框架全景.html) — 视觉化全景（双击打开，手机也能看，**主人首选入口**）
- 📋 [方案设计.md](方案设计.md) — 完整方案文档（细节、奴婢与主人议定全过程）

## 核心定位（一句话）

> 主人新方向通过 S1（gstack 八问）站住后 → 进入本框架的"需求池" → 全自动跑五环节 → **唯一硬卡点：主人合入上线确认** → 上线后 4A 活文档自增。

## 接入方式（v0.1 待打磨）

业务项目 `CLAUDE.md` 顶部加：

```
@~/Desktop/AILab/Agent驱动研发框架/CLAUDE.md
```

接入后业务项目获得：

- 六环节流程定义（需求 → 分析 → PRD → 4A 增量 → 研发 → 测试 → 主人合入）
- 4A 架构模板（业务 / 应用 / 数据 / 技术）
- 产物模板（需求卡、PRD、4A 增量、测试报告、合入确认）
- Agent 协议（需求分析 / PRD 设计 / 4A 增量 / 研发 / 测试）

## 与现有体系的关系

| 现有体系 | 本框架的关系 |
|---|---|
| S1-S5 五段矩阵 | 本框架仅覆盖 S2 末 → S5；S1 不动 |
| gstack | 思维底座不动，新方向先过八问 |
| superpowers | 研发 Agent 通过 TDD / 调试 / worktrees 落地 |
| Playwright Agents | 测试 Agent 通过 Planner / Generator / Healer 三件套落地 |
| HTML 优先输出范式 v1.0 | 完全兼容，本框架是其落地实例 |
| mkdocs（`_view/`） | 持续渲染层，本框架接入项目自动得 |
