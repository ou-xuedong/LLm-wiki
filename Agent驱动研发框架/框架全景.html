Agent 驱动研发框架

AILab 体系的研发执行框架。被业务项目读取，让"需求 → PRD → 研发 → 测试 → 上线"形成可追踪、可迭代、人在环的自动化流水线。

核心定位

主人新方向通过 S1（gstack 八问）站住后 → 进入本框架的"需求池" → 全自动跑五环节 → 唯一硬卡点：主人合入上线确认 → 上线后 4A 活文档自增。

📋 需求 ID 追踪链

每条需求一个唯一 ID（如 R-2026-001），所有产物用 ID 串起——可追溯、可下钻、可维护。 

需求池/R-2026-001_用户登录/
├ 0_需求卡.md          # 原始需求
├ 1_分析报告.md         # 环 1 产出
├ 2_PRD.md / .html      # 环 2 产出
├ 3_4A增量.md           # 环 3 产出
├ 4_代码.md             # 环 4 产出（git 链接）
├ 5_测试报告.html        # 环 5 产出
├ 6_合入确认.md          # 🔴 主人改 status
└ status.yaml          # 当前状态
