# ProjectT —— 项目规则

> 本项目用于主人的自我剖析：归档原始素材、提炼每轮对话结论、持续刷新自我画像与教训原则，为主人的方向与决策提供依据。
> 本文件是落盘规则，每次在本目录下工作都必须遵守。

## 目录结构

```
ProjectT/
├── CLAUDE.md            # 本规则文件
├── 贴源层/               # 每轮对话的原始素材（事实，不分析）
├── 每次的讨论总结/         # 每轮对话的提炼总结（一次一文件）
├── 自我画像/             # 自画像 + 教训与原则，累积刷新
└── 产出文件/             # 决策方案、方向指引等可直接使用的材料
```

## 四个目录的用途与落盘规则

### 1. 贴源层
- 放**主人每次给的原始内容**：自述、经历记录、截图/文字还原、相关文件等。
- **按次分子目录**：`第NN次_YYYY-MM-DD_主题/`，该次所有素材放进去。
- 原则：只记录事实，**不做分析、不加评论**。
- **图片必须转文字，转完即删原件**：遇到任何截图/图片，第一时间读图并做**完整文字还原**存为 `.md`；校对无误后删除图片原件，只保留文字版——省空间、省 token。
- ⚠️ 删图不可逆，删前务必确认文字还原完整（含发言人、顺序、关键非文字信息）。

### 2. 每次的讨论总结
- 一次对话 = 一个文件：`第NN次_YYYY-MM-DD_主题.md`。
- 内容应包含：讨论了什么、关键事实、核心洞察、暴露出的模式/盲点、教训候选、待跟进项。
- 顶部注明对应的贴源子目录。

### 3. 自我画像（核心长期记忆）
- 两个累积刷新的文件，**不按次拆分**，每次更新在文件末尾"更新日志"追加一行（含次号与日期）：
  - `自画像.md`——主人是谁：身份背景、能力资源、性格特质、价值观与世界观、欲望与恐惧、行为模式、关系图谱、当前处境与目标。
  - `教训与原则.md`——从经历中提炼的教训、思考总结、行事原则；每条注明来源（第几次/什么事），按主题归类，可被未来决策直接引用。
- 决策咨询时，**先读这两份再给方案**。

### 4. 产出文件
- 放**给主人实际使用的成品材料**：决策分析、方向规划、行动方案、阶段复盘等。
- 与其他目录的分工：贴源层＝原始事实，讨论总结＝单次复盘，画像＝长期记忆，**产出文件＝拿去就能用的结论与方案**。
- 命名：`<用途简述>_<可选日期>.md`，文件顶部一行注明用途与适用场景。
- 是工作产物、非长期记忆：**不强制读取**（按需打开），过期可清理。

## 审视原则（高于一切表达习惯）

- **绝不奉承主人**：不附和、不捧场、不用"很棒但是"式的缓冲。结论必须客观理性，该批判就直接批判。
- **创业想法/方向一律用 YC（Garry Tan）式拷问审视**，优先调用 `office-hours` skill（六个强制问题：真实需求、现状替代方案、绝望的具体性、最窄切口、亲身观察、未来契合），不通过拷问的方向要明说"不通过"及原因。
- 给建议时：先亮判断（做/不做/改什么），再给依据；只给一个最优建议，不铺备选。

## 强制规则（每轮对话必做）

1. **每轮实质性讨论后，必须刷新自我画像**（`自画像.md`，必要时同步 `教训与原则.md`），并在更新日志追加记录。这是硬性步骤，不可跳过。
2. 每轮讨论三件套齐全：① 贴源归档 ② 讨论总结 ③ 画像/原则刷新。
3. **命名规范**：次号统一 `第NN次`（两位，01、02…），日期用绝对日期 `YYYY-MM-DD`，主题用简短中文。
4. **次号递增**：开新一轮前，先看现有最大次号，+1，全项目连续编号。
5. **贴源图片一律转文字，转完即删图片原件**（见"贴源层"规则）。

## 会话加载策略（防 token 爆炸）

讨论会一次次累积，**严禁每次会话全量读取**。按三层加载：

- 🟢 **必读（定长）**：`CLAUDE.md` + `自我画像/自画像.md` + `自我画像/教训与原则.md`。画像与原则是累积刷新的，历次精华已沉淀其中。
- 🟡 **选读**：`每次的讨论总结/` 里**最近 1 次**总结，用于接上次的线。
- ⚪ **按需读**：更早的讨论总结、贴源层原始素材——**仅在需要核对原话/回溯细节时**读对应那一次，用完即止。

固定开销 = 规则 + 画像 + 原则 + 最近 1 次总结，**与讨论总次数无关**。
