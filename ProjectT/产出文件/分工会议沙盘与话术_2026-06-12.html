明日分工会议 · 沙盘与话术

🎯 全场只有一个目标：分工表上你名字旁写着"Agent 自进化机制预研"（措辞抽象、无 caseBank 等模块名、交付物为预研报告）——既挡住 DataOS，又不背牵头与节点。
情报修正

R博刚被松波骗过，明天他不是平常心

松波两周前承诺不走→保住 A→拿完年终奖→提离职。这意味着 R博 进会议室时：①更急于把人和承诺白纸黑字锁死（"代管/先兼着"压力升级）；②对"听安排"式的低承诺表态更敏感，会嗅"下一个要跑的人"。
对策拒角色的姿态要比原稿更"给抓手"：每个拒绝后面都跟一个具体的、马上兑现的小承诺（月度书面预研同步、材料把关、方案评审）——让 R博 手里有东西，他才不需要用名分锁你。
明天绝不提去年绩效 B 的委屈，绝不提松波保 A 的操作——前者读作讨价，后者读作教 R博 防你。
第0步

会前两步棋 + 一份功课（今天必做）

已知情报：小心腹拟给 R博 的人员安排里，你仍在 DataOS。当众掰一个预设好的方案，成本远高于会前改表——分工会议宣布的多是会前已定的事。
① 最急：把"跟舒潘吴迪一起开发"变成真的已对小心腹说过"在和舒潘吴迪一起开发自进化系统"——这句一捅就破（R博一句话即可核实）。今晚/明早给舒潘、吴迪发接口设计文档："我这边自进化系统的数据接口设计出来了，发你看下，约个时间对一下系统对接。"一条消息把谎话变成有时间戳的事实，明天还能说"正在和他们对接口"。
② 找小心腹（不问"我要不要参加"，问 R博 的反应；不再翻那张表的旧账）"明天分工会前想跟你对个信息：我之前提的转开发和自进化系统那事，R博 后来有什么说法没？我想提前准备我那块的介绍，免得会上跟他对不上。"——探到的是唯一未知数（R博 的态度），姿态是"准备充分"而不是"想躲会"。
③ 争取会前与 R博 单聊 10 分钟（约不到就微信/邮件递话）"两个事提前跟您对齐：DataOS 需要一个全职 owner 跟客户和汇报节奏，建议安排能全职扑上去的人；我手上 Agent 自进化预研正在机制验证期，我想把它作为我的分工正式立项，按月给您书面结论。方案评审和材料把关我照样支撑。"
④ 今晚功课（2-3 小时）把大模型生成的接口设计自己从头过一遍并校验——发给舒潘吴迪之前必须先过（发出去的东西就是你的名片），明天能就细节脱稿讲 5 分钟。占坑的可信度不靠进度，靠"对细节的掌握程度"。松波事件后，R博 对"空心项目"的嗅觉正在峰值。
"Agent 自进化"至今是你单方面的说法，未经正式确认——明天等于请 R博 当众首次批准它，而他手里的表写着 DataOS。不做会前动作，占坑成功率减半。
全程行为守则（比话术更重要）
  • 后发言——分工类会议先开口的人先背锅，等格局清楚再认领自己那块
  • 只谈事，不谈人——不荐人、不评松波、不议裕钦，组织话题一律"听安排"
  • 主动给措辞——分工表落字时抢着提供自己条目的写法，别让别人替你写
  • 承诺只到"预研报告"——任何"系统上线/demo 日期"都不接

一、高概率 · 必须备稿

P0

场景1：R博点名你牵头（或"代管"）大模型产品

概率：高
对方可能说："裕钦和松波都要走了，这块还是你最合适，你来牵头吧。"　变体：先戴高帽（"BP/SP 都是你做的"）再点名；或包装成"临时代管，过渡一下"。
话术"我认真想过这个事。我的产出峰值在纵深做产品和预研，牵头要花大量时间做协调和向上同步，那是把我用在产出最低的地方——对部门是亏的。方案、评审、汇报材料我可以全力支撑牵头人，这块我不含糊。"
若对方坚持"先代管过渡""过渡期我可以把材料和方案口径都兜住，但'代管'这个名分别落在我头上——落了名分就是责任主体，反而耽误交接节奏。"
不说真实原因（不想向董事长汇报/要走）；不推荐任何接任人选；不当场把话说死到"永远不接"。
"代管"是最危险的变体——临时名分会自动转正。用"支撑牵头人"给足姿态，把拒绝包装成为部门省钱。
P0

场景2：分工表当场落格子，轮到你

概率：极高
对方可能说："你这块写什么？什么时候能看到东西？"
话术（主动报措辞）"我这块写'Agent 自进化机制预研'：验证知识驱动的 Agent 能不能在用例中自我迭代。预研阶段交付物是阶段性预研报告和可行性结论，我按月同步一次结论。"
被追问上线/demo 时间"预研的承诺要拿验证结果说话，现在给系统时间点是误导规划。月度同步里我会给'继续投/转向/停'的明确建议，这比拍一个日期负责任。"
被问"进展到哪了/怎么还没跟舒潘洪彬对接""目前完成了数据接口设计，正在校验。机制没验证通过之前拉舒潘、洪彬对接是浪费两边时间——预研结论出来、确认值得做，才到系统对接阶段，那时我主动找他们。"（前提：今晚把接口设计真的过完）
条目里不出现 caseBank、记忆库、案例库等与创业产品文档重名的模块词；不接"Q3 上线"类节点；不被追问出"还没认真看过设计"。
月度同步预研结论成本极低、完全可控，却能堵住"没产出"的质疑——这是用最便宜的承诺买最大的自由。
P0

场景3：DataOS（或同类无沉淀大活）塞给你

概率：极高（拟稿表已写你）
对方可能说："DataOS 产品这块你最熟，松波走了就你能接。"　注意：这大概率不是临时起意，而是宣读已拟好的安排——所以第0步的会前动作才是主战场，会上是补救。
话术"DataOS 需要一个全职投入、持续跟客户和汇报节奏的人，我手上预研正在关键验证期，两头接等于两头都做砸。我可以做 DataOS 的方案评审和材料把关，owner 得是能全职扑上去的人。"
若被强压"那我建议明确优先级：要我全职接 DataOS，预研就得停——这个取舍请 R博拍板，我都执行。"
不说"DataOS 没价值"（否定的是 R博辖区）；不答应"先兼着"。
"兼着"是温水煮蛙的入口。把取舍抛回给 R博：让他亲手停掉自己认可过的 Agent 预研，多数领导不会。

二、中概率 · 备口径

P1

场景4：要你向董事长汇报（或"下次你来讲"）

概率：中
话术"材料我来出、口径我来磨，这是我擅长的；但汇报出口建议收在牵头人一个口，多头汇报董事长那边信息会打架。"
把材料能力变成挡箭牌：给足支撑姿态，同时让别人站在汇报炮口上。DataOS 副总监的前车之鉴说明那个位置是负资产。
P1

场景5：当场要看 Agent 系统演示

概率：中
对方可能说："你那个自进化系统跑起来了吗？给大家看看。"
话术"现在跑的是机制验证实验，不是产品形态，拿半成品演示只会引导出错误预期。第一份预研报告出来我主动约时间讲，有数据有结论。"
绝不展示任何创业项目代码/界面，哪怕"长得差不多"。一次展示＝公司版与创业版从此说不清。
P1

场景6：给你配人一起做预研

概率：中低
对方可能说："这个方向不错，给你配两个人加快一点。"——听着是好事，实际是把你变成 team leader：带人＝管理＝汇报＝节点。
话术"预研期加人是浪费，问题还没收敛，人多了只能等我。等验证出结论、进入工程化阶段，我第一个来要人。"
公司同事一旦实质参与，既把你拖回管理角色，也让"思路同源"的边界更难说清。

三、低概率 · 留心即可

P2

场景7：松波交接，把活倒向你

概率：低中
交接安排时："这几块松波的活，先转给最熟的人。"　或松波当众旧事重提"本来说好你主导"。
接交接"交接文档我可以 review 把关，确保不丢东西；接手 owner 定了之后我配合他熟悉。"
接旧事"那是当时的讨论了，现在按 R博的分工框架走就行。"（一句带过，不解释当时为什么拒）
只接"review 文档"，不接"暂时先管着"。
P2

场景8：试探站队/去留（"人都走了，你怎么想？"）

概率：低
话术"我关注的是把手上预研做出结果，组织上的事我听安排。"
不表忠（过度表忠＝被委以重任的前奏），不露离意，不评价裕钦/松波的去留。
P2

场景9：征求你对整体分工方案的意见

概率：低中
话术"我就提一条：按业务线切，一条线一个 owner，权责和考核对齐，别出现双头。具体人选 R博定就好。"
给一条只关于"结构"的意见，显得在状态又零站队成本；说完就收，不展开。

四、口袋里的三句话（万能兜底）

  • 挡角色："我产出最大的位置是纵深做产品，协调牵头是把我用在产出最低的地方。"
  • 挡节点："预研的承诺要拿验证结果说话，我按月给结论，不拍日期。"
  • 挡表态："事上我有观点，人和组织的事我听安排。"
任何没预料到的问题，先归类到"角色 / 节点 / 表态"三类之一，套对应那句，再视情况补一句细节。当场拿不准的，一律："这个我下来想清楚再给你答复。"——把战场拖回自己的时间。
