📐 Schema 规范

Wiki 遵循 SCHEMA.md 定义的 AI/创业情报领域规范。核心要点：

规范项要求
文件命名全部小写，中划线分隔，无空格
Wikilinks每个页面至少 2 个 outbound links
Tags必须含 type 标签（kols/vc/company/concept/query/event）
Confidence高/中/低，反映信息可信度
Provenance综合 3+ 来源时加 ^[raw/articles/...] 标注
冲突处理新旧矛盾 → 保留双方，标 contested: true

📋 Frontmatter 必填字段

---
title: 页面标题
created: YYYY-MM-DD          # 首次创建日期
updated: YYYY-MM-DD          # 最近更新日期
type: kols | company | vc | event | concept | query | comparison
tags:
  - #领域标签
  - #类型标签
# 可选：
confidence: high | medium | low
sources: [raw/articles/xxx.md]
contested: true
contradictions: [other-slug]
---
Tag 体系： #kols#vc#company#event #agent#memory#rag#inference #ecommerce#amazon#framework #methodology#principle

🔗 Wikilinks 用法

同目录引用

[[target-page]]
[[target-page|显示文字]]

跨目录引用

[[entities/kols/sam-altman|Sam Altman]]
[[concepts/agentic-engineering]]
[[queries/2026-03-28_ai日报关键信号洞察]]

Relations 字段展开规则

旧知识库的 relations.* 字段自动展开为正文中的 ## 关联 段落，用 **标签** 格式呈现：

## 关联

**所属**：[[cursor-ai编程智能体]], [[openai-大模型]]
**专长**：[[vibe-coding]], [[agentic-engineering]]
**相关 KOL**：[[andrej-karpathy]], [[geoffrey-huntley]]

🔄 日常工作流

新增情报

  1. 摄入原始材料到 raw/articles/（不改原始文件）
  2. 创建/更新 entities/、concepts/ 或 queries/ 中的 wiki 页面
  3. 使用 [[wikilinks]] 关联相关页面
  4. 更新 index.md 中的对应 section
  5. 追加 log.md（格式：## [YYYY-MM-DD] action | subject）

更新现有页面

  1. 新信息与旧内容冲突 → 保留旧内容，追加新内容
  2. 添加 contested: true + contradictions: [other-slug]
  3. 报告给用户（lint 环节）

归档

内容完全被取代时 → 移入 _archive/，从 index.md 移除，不删除（保留证据）

🔍 qmd 检索
检索 wiki 内容 必须且只能用 qmd CLI。禁止用 grep/find/Glob 替代。

安装 qmd（如未安装）

npm install -g @tobi/qmd

配置 qmd 指向新 wiki

# 在 ~/.hermes/.env 中配置：
WIKI_PATH=/Users/ouxuedong/Desktop/AILab/LLm-wiki

# 或通过 qmd CLI：
qmd init --path ~/Desktop/AILab/LLm-wiki

检索命令

操作命令
关键词搜索qmd search "关键词"
自然语言问答qmd query "Sam Altman 最近有什么动态"
MCP 模式通过 MCP 工具调用 qmd（优先）

🛠 qmd MCP 工具注册

已在 ~/.hermes/.env 中配置：

WIKI_PATH=/Users/ouxuedong/Desktop/AILab/LLm-wiki

重启 Hermes Agent 后，MCP 工具 qmd_mcp 自动可用，支持：

  • qmd_search — 关键词全文搜索
  • qmd_query — 自然语言问答检索
  • qmd_list — 列出所有 wiki 页面
qmd MCP 工具的 wiki-path 参数默认为 WIKI_PATH 环境变量，无需每次传入。
