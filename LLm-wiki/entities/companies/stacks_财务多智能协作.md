# Stacks（财务多智能体协作平台）

## 1. 核心定位与业务模式
Stacks 定位为 **"财务团队的 AI Agent 协作平台"**，通过多个专业 AI Agent 协同处理跨系统的财务对账、报表、审计等复杂任务。与传统 RPA 不同，Stacks 的 Agents 之间可以通信、共享上下文、协调决策，真正实现跨孤岛自动化。

**核心问题**：企业财务数据分散在 ERP、CRM、Excel、银行系统、发票系统等多个孤岛，Stacks 用多 Agent 协作解决跨系统对账难题。

## 2. 护城河 (Moat)
- **多 Agent 协作架构**：自研的多 Agent 通信和协调机制，工程壁垒较高
- **财务场景专精**：不像通用工具那样浮于表面，真正理解复式记账、财务报表、合规要求
- **高客户粘性**：一旦财务团队在 Stacks 上跑月结/对账流程，切换成本极高
- **顶级 VC 支持**：由 a16z、Sequoia 等顶级机构支持，资源充足

## 3. 命门与软肋 (Weaknesses)
- **ERP 厂商压制**：SAP、Oracle 等传统 ERP 厂商可能推出同类功能，形成正面压制
- **实施复杂度**：企业财务系统定制化程度高，集成成本高，Sales Cycle 极长
- **AI 幻觉在财务场景更危险**：数字一旦出错，后果严重，企业接受度有上限
- **竞争激烈**：Ramp、Brex、Mesh 等财务 AI 公司都在抢夺同一批客户

## 4. 重大事件与动态
- [2026] 获顶级 VC 支持，持续扩展多 Agent 协作能力
- [2025] 重点拓展中型企业市场，推出 SMB 版本
