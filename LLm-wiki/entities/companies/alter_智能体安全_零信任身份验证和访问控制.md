# Alter（AI Agent 零信任身份与访问控制）

## 1. 核心定位与业务模式
Alter 专注于 **AI Agent 时代的零信任（Zero Trust）身份与访问控制**。当 Agent 需要以某个用户身份访问多个系统时，Alter 提供动态的、最小权限的身份验证机制——确保 Agent 只能访问它被授权的最小数据集，且每次访问都经过实时风险评估。

**核心问题**：Agent 时代，身份不再只是"人"，还包括"AI Agent"。如何给 AI Agent 发数字身份、如何管理它的权限、如何审计它的行为，是企业级 AI 落地的关键安全课题。

## 2. 护城河 (Moat)
- **零信任与 AI Agent 的交叉**：这是新出现的细分品类，Alter 是最早的专注者之一
- **动态风险评估**：实时评估 Agent 请求的风险等级，动态调整权限，不是静态规则
- **最小权限执行**：确保 Agent 只能访问任务所需的最少信息，降低数据泄露风险
- **合规天然适配**：满足 SOX、GDPR、EU AI Act 等对 AI 决策可审计性的要求

## 3. 命门与软肋 (Weaknesses)
- **市场太新**：企业对"AI Agent 身份管理"的需求是否真的成熟，还是在预研阶段？
- **平台方挤压**：Okta、CyberArk 等传统身份管理公司可能快速推出类似功能
- **缺乏公开融资信息**：公司规模和融资情况不明，难以判断长期竞争力
- **技术验证周期长**：安全产品的企业验证周期很长，销售效率低

## 4. 重大事件与动态
- [2026] 持续深耕 AI Agent 身份管理赛道
- [2025] 产品 GA，开始服务首批金融和企业客户
