# 🏢 Warp (AI 终端 / Agent 开发环境)

## 1. 核心定位与业务模式
Warp 是一款基于 Rust 编写的现代化终端，定位为"Agentic 开发环境"（ADE）。核心逻辑：从"人手动操作工具"转向"自然语言驱动 AI 代理执行"。商业模式：AGPL 开源核心 + Oz 闭源云代理平台商业化。

**关键数据：**
- 2022 年产品启动，2026 年 5 月 1 日全面开源
- 累计融资：7300 万美元（B 轮 5000 万美元）
- 背后投资人：Sam Altman、Salesforce CEO、LinkedIn CEO、Figma CEO
- 全球服务工程师超 70 万（2026 年已达 50 万+）

## 2. 护城河 (Moat)
- **Rust 性能优势**：GPU 加速、基于块的输出，性能远超传统 bash/zsh
- **AI 原生集成**：非插件式叠加，从设计之初就将 AI 作为一等公民
- **范式卡位**：从"终端工具"进化为"可被 AI 操控的操作层"，占位 Agent 执行入口
- **开源生态**：AGPL 开源核心，Oz 闭源变现，开源获流量、ToB 商业化

## 3. 命门与软肋 (Weaknesses)
- 商业化路径：Oz 闭源层是否被社区接受是关键风险
- 定位转换挑战：从"终端工具"到"Agent 开发环境"，用户心智切换成本
- 竞争压力：VS Code/IDEA 仍是主流，Claude Code/Cursor 快速迭代

## 4. 重大事件与动态
- [2026-05-01] **全面开源**：GitHub 开源（AGPL），15 小时斩获 3.5 万 Star，OpenAI 成为创始赞助商，GPT 系列驱动 Warp 代理管理工作流
- [2026-05-01] 新增支持 Kimi、MiniMax、Qwen（通义千问）等开源模型，上线"自动（开放）"模型路由
- [2026-04-29] 开源事件引发社区讨论（详见 [[warp_开源事件与ai终端演进洞察]]）
- [2025-06] Warp 2.0 发布，向 Agentic 开发环境转型

## 5. 与主人方向的关联
**战略参考价值极高：**
1. **商业模式参照**：开源获流量 + 闭源 Oz 商业化 → 主人可考虑"开源投流模板 + 闭源 Agent 执行"模式
2. **产品架构参照**：OpenSpec + Superpowers + OMO 三层架构 → 投流助手可借鉴：规范层（策略约束）/ 纪律层（自动测试）/ 沉淀层（历史复盘）
3. **定位参照**：Warp 占位"执行入口"，主人投流助手可占位"投流决策入口"
