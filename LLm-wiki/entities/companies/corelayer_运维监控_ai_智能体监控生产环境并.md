# Corelayer（AI Agent 驱动的生产环境可观测性平台）

## 1. 核心定位与业务模式
Corelayer 定位为 **用 AI Agent 监控生产环境的可观测性平台**，核心能力：AI Agent 7x24 监控应用和基础设施，实时发现异常、诊断根因、自动触发修复。与传统可观测性工具（Datadog、Splunk）依赖规则和阈值不同，Corelayer 的 AI 能理解正常行为的上下文，判断是否真的出问题，并执行修复动作。

**核心问题**：现代分布式系统故障排查极其复杂——日志、指标、链路追踪分布在几十个工具里。Corelayer 的 AI Agent 帮 SRE 把这些信息聚合理解，给出"哪里出了问题+为什么+怎么修"的结论。

## 2. 护城河 (Moat)
- **AI Agent 原生**：监控和诊断本身就是一个 Agent 在运行，能执行修复动作，不是纯告警工具
- **全链路覆盖**：从应用到基础设施、从日志到链路追踪全覆盖
- **自动化修复闭环**：检测+诊断+修复，是业界少数能完成完整闭环的平台
- **现代化架构**：面向云原生和 Kubernetes 世界，比 Splunk 等传统工具更适应现代技术栈

## 3. 命门与软肋 (Weaknesses)
- **Datadog/Splunk 竞争**：两家巨头都在加 AI 可观测性功能，Corelayer 必须找到足够差异化的点
- **误报率挑战**：AI 异常检测的误报直接影响 SRE 对平台的信任，平衡难度大
- **企业安全要求**：生产环境监控涉及敏感数据，企业对供应商的安全审查极严
- **冷启动问题**：没有足够多的客户案例和基准数据，AI 模型的准确性受限

## 4. 重大事件与动态
- [2026] 持续迭代 AI 根因分析和自动修复能力
- [2025] 产品 GA，开始服务第一批云原生企业客户
