# Sycamore（企业 Agent 安全调度平台）

## 1. 核心定位与业务模式
Sycamore 是**企业级 AI Agent 安全调度与编排平台**，核心解决"企业在赋予 Agent 执行权时的信任与安全问题"。与 Temporal 的技术导向不同，Sycamore 更偏向安全与合规：企业需要一个可信的中间层来监管、调度、审计所有 Agent 的行为。

**核心问题**：当企业把跨系统的操作权限（如 ERP 审批、数据查询、邮件发送）交给 AI Agent 时，谁来保证 Agent 不会出错、不会被攻击、不会被滥用？

## 2. 护城河 (Moat)
- **安全合规双轨**：零信任架构（Zero Trust）+ 合规审计双轨，是进入大型企业的通行证
- **多系统集成**：支持 SAP、Oracle、Salesforce 等主流企业软件的 Agent 调度，工程壁垒高
- **VC 背书**：红杉、光速等顶级 VC 早期大手笔投入，品牌和技术资源双重加持
- **先发优势**：企业 Agent 安全调度是一个新兴品类，Sycamore 是最早的玩家之一

## 3. 命门与软肋 (Weaknesses)
- **技术护城河不够深**：如果微软、Salesforce 等平台方自己推出同类功能，Sycamore 的中间件价值可能被取代
- **销售周期长**：企业安全产品销售周期极长，现金流压力大
- **Agent 标准不统一**：不同厂商的 Agent 接口不同，集成成本高，扩展速度受限
- **Temporal 的竞争**：Temporal 在技术层也在做"安全调度"，两家的边界正在模糊

## 4. 重大事件与动态
- [2026] 持续扩展对 MCP、A2A 等新 Agent 通信协议的支持
- [2025] 获红杉中国或光速创投新一轮融资，加速企业市场渗透
- [2024] 产品 GA，开始服务第一批财富500强企业
