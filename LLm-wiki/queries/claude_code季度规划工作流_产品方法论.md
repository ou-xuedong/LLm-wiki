# 💡 Claude Code 季度规划工作流：4 角色分工法

## 核心结论

做季度规划的**正确顺序**是：先 OKR → 再竞品 → 最后路线图。把这套流程在 Claude Code 中固化成 4 个固定角色，让 AI 承担分工而不承担判断，人负责战略裁剪。

## 详细分析

### 顺序铁律：OKR → 竞品 → 路线图

大多数团队季度规划的典型错误：
- 先拉需求 → 再找理由 → 最后补战略
- 结果：路线图 = 需求大杂烩 = 不同人焦虑的排期表

正确顺序的逻辑：
1. **先确认这季度到底要赢什么**（OKR 定靶）
2. **再判断外部环境发生了什么**（竞品扫描）
3. **最后才决定资源投到哪里**（路线图生成）

### 4 角色分工（Claude Code 工作流）

| 角色 | 职责 |
|---|---|
| OKR 分析员 | 把公司目标翻译成产品目标，识别真假优先级 |
| 竞品情报员 | 扫描竞品动作，提炼值得跟、不该跟、必须防的点 |
| 路线图设计员 | 把目标、资源、依赖关系收束成一份季度路线图 |
| 汇报打磨员 | 把分析结果整理成老板能快速拍板的表达结构 |

### AI 不是替你做战略，而是高配实习生

关键定位：**Claude Code 不是替你做判断的人，而是你带的 4 个高配实习生。**

- 你负责：定方向、做判断、拍板
- 它负责：脏活、累活、重复活、结构化活

这样人真正花时间的地方不再是"搬运资料"，而是"做判断"。

### 三步启动建议

1. 先固定季度规划的**输入清单**（老板方向、销售声音、运营诉求、研发资源）
2. 先把 3 个固定**产出模板**做出来（OKR 拆解模板、竞品扫描模板、路线图模板）
3. 别追求一步到位，先追求**每个季度都能复用**

## 数据与证据

- 文档写 50 页、竞品截图 80 张、OKR 写得很漂亮 → 但三个月后团队还是问"我们这季度到底为什么做这些？"
- 根源不是不努力，是**不成系统**

## 行动建议

对于主人做投流 Agent 的启发：
- 投流决策流程也可借鉴此**先定目标 → 再看竞品数据 → 最后生成投放策略**的顺序
- 可把 AI 定位为"投流执行层的高配实习生"，专注执行与数据整理，人类专注策略判断
- 固定输入清单和产出模板，实现季度/月度复用
