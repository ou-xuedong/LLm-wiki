# 💡 架构范式转移：Agentic Search 正在取代静态 RAG

## 1. 核心论点

过去一年"RAG 已死"的论调甚嚣尘上。然而剖析 Claude Code、Codex 等新一代 Agent CLI 的底层架构后，**死的并非 RAG 宏观范式，而是"万物皆需 Embedding 与向量库"的静态预索引执念**。

在代码场景下，由 LLM 驱动的系统级暴力搜索（Grep），正以一种极致的工程实用主义回归。

## 2. 核心洞察：锚点即答案

代码中的**类名、函数名本身就是工程师预埋的高精度锚点**：
- 工程师在命名时已经编码了大量语义信息
- 与其花算力猜测这些锚点的"多维向量距离"
- 不如构建足够强大的外围 Harness，让有深厚编程逻辑的 LLM 自己拿起 Grep，精准出击

## 3. 两大架构范式

### 范式 A：零索引 + Grep（Claude Code 模式）

**架构设计**：
- 工具链下放：ripgrep、Glob、FileRead 直接暴露给 LLM
- 探索即循环：LLM 自主决定搜索关键词和模式，多轮迭代
- 软引导代替硬约束：不预设固定流水线

**底层性能支撑**（让零索引成为工程现实）：
- SIMD 向量化：CPU 字节级并行比较
- mmap 零拷贝：大文件内存映射
- Page Cache：本地项目常驻内存
- 实测：百万行代码并发正则扫描 < 100ms

**适用场景**：中小型代码库（MB ~ 数百MB）

### 范式 B：混合双索引（Cursor 模式）

**架构设计**：语义 Embedding + Trigram 倒排，离线预建索引

**适用场景**：百人协作项目、GB 级别超大代码仓库、跨项目知识域检索

## 4. 企业落地的三大 Token 成本管控机制

| 机制 | 效果 |
|------|------|
| Prompt Cache | 重复前缀成本降低 80%+ |
| Context Auto-compaction | 逼近阈值时强制压缩摘要 |
| Sub-agent 隔离 | 中间日志自行消化，只回传精炼结论 |

## 5. 三条本质规律（来自玄姐）

**规律一**：AI产品的上限由模型能力决定，不是产品设计能力。

**规律二**：工作流深度，是比模型能力更可持续的护城河。
> 当前商业化最成功的 AI 产品（如 Copilot），核心竞争力不是模型本身——模型可替换——而是对特定工作流的深度嵌入和用户数据的持续积累。

**规律三**：终局是 Agent，但演进是分阶段的，不要跳步。
> 短期 Copilot → 中期工作流自动化 → 长期高可靠性自主执行 Agent

## 6. 对主人方向（亚马逊投流 Agent）的映射价值

- **Sub-agent 隔离** → 多账号投流管理：主控 Agent 派生子 Agent 处理单账号，隔离 Context
- **Context 压缩** → 长对话历史自动摘要
- **Token 成本管控** → API 调用成本优化
- **Harness 思路** → 投流 Agent 的核心竞争力在于"运行环境设计"，而非盲目追新模型
