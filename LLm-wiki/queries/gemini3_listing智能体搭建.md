# 💡 Gemini3 Listing智能体搭建：Rufus级爆款文案工作流

## 核心结论

基于Google Gemini 3构建的亚马逊Listing撰写智能体，通过**A9+COSMO+Rufus三维算法融合**，实现"30秒产出符合平台算法偏好的爆款Listing"。其核心价值在于**将亚马逊运营隐性知识（合规红线+算法偏好+竞品破绽）结构化注入工作流**。

## 一、为什么需要Rufus级Listing智能体？

**三代亚马逊算法对Listing的要求演进**：
- **A9时代**：词堆够了就有流量（关键词密度为王）
- **COSMO时代**：场景对了才转化（意图-属性-场景三元组）
- **Rufus时代**：参数精准才被检索（Rufus是事实型AI，不吃形容词）

**结论**：一套Listing要同时满足三代算法的要求，手工撰写已无法 scale，必须依赖AI。

## 二、智能体架构（四部分）

| 组成部分 | 作用 | 示例 |
|----------|------|------|
| **名称** | 后台管理与快速识别 | "AMZ那些事｜亚马逊Listing撰写专家" |
| **说明** | 告知用户上传哪些文件 | 上传4个文件 + 基于A9/COSMO/Rufus生成 |
| **指令(Instructions)** | 智能体的执行逻辑（大脑） | 三步工作流：数据分析→差异化撰写→合规自查 |
| **知识库(Knowledge)** | 强制遵守的合规红线 | amazon_compliance_blacklist.txt |

## 三、输入文件规范

智能体依赖4个结构化输入：

1. **竞品出单词报告.csv** — 分析流量来源，找用户真实搜索词
2. **ABA关键词数据.csv** — 决定埋词权重，锁定高价值词
3. **竞品Listing文本.txt** — 多头部竞品文案+Review优缺点，进行市场格局分析
4. **本品属性表.txt** — 精准参数，喂饱Rufus算法的"事实饥饿"

## 四、三步工作流

### Step 1：多维数据清洗与市场格局分析
```
Rufus属性提取 → 读取本品属性表，提取所有具体参数
多竞品格局分析 → 找共性(Parity)+找缺口(Gap)
COSMO场景映射 → 结合竞品出单词，锁定用户真实场景
A9关键词分级 → 基于ABA数据，锁定高权重核心词
```

### Step 2：差异化Listing撰写
**标题公式**：`[品牌] + [ABA核心大词] + [针对竞品痛点的核心改良点] + [COSMO场景] + [属性]`

**五点描述格式**：
- Point 1：痛点狙击（针对竞品共同缺陷）
- Point 2：场景沉浸（竞品未充分挖掘的COSMO场景）
- Point 3：硬核参数（Rufus友好格式）
- Point 4：适用人群/礼物场景
- Point 5：售后/信任背书

**产品描述**：HTML排版 + 详细参数表 + FAQ（FAQ中专门回应竞品普遍差评）

### Step 3：合规性与侵权自查
对照知识库中的黑名单，执行三重扫描：
1. 促销词扫描（Best seller, Free shipping等）
2. 侵权/敏感词扫描（竞品品牌名、Chaps stick等）
3. 夸大宣传扫描（所有参数必须来自本品属性表）

## 五、亚马逊Listing五大合规雷区（速查）

| 雷区类型 | 严禁词示例 | 建议替代 |
|----------|-----------|----------|
| 促销/主观评价 | Best seller, Free shipping, #1 | 客观描述性能参数 |
| 知识产权/品牌兼容性 | "iPhone Case" | "Case compatible with iPhone" |
| 医疗器械/功效 | Cure, Heal, Prevent, FDA approved | 禁用疾病相关表述 |
| 杀虫剂/抗菌 | Anti-bacterial, Non-toxic, Disinfect | 改为具体材质描述 |
| 环保声明 | Eco-friendly, Biodegradable, Green | 使用具体材质描述 |

## 六、与投流决策助手的协同

> Listing智能体负责"上车后"的内容呈现  
> **投流决策助手负责"上车前+上车时"的广告决策**  
> 两者是亚马逊卖方运营的前后道配合

**潜在整合方向**：Listing智能体输出的"竞品分析洞察+差异化策略"可作为投流决策助手的输入，形成"选词→写词→投词"的闭环。
