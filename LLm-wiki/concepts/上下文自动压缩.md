# 🗜️ 上下文自动压缩（Context Auto-compaction）

## 1. 定义

**Context Auto-compaction** 是 Agentic Harness 的一种核心机制：当累计 Token 数逼近 Context Window 阈值时，强制触发摘要生成机制，将早期的搜索试错轨迹压缩为精炼结论，为后续操作腾出空间。

## 2. 问题背景：Token 爆炸

多轮试错搜索场景下，无用的代码片段和中间检索日志会持续倾倒进 Context Window：

- 拖慢模型响应速度
- 产生难以承受的 API 费用
- 稀释真正有价值的信息密度

## 3. 机制原理

```
[多轮搜索轨迹原始数据]
    ↓ 逼近 Token 阈值
[触发摘要生成]
    ↓
[精炼结论摘要] + [保留关键上下文]
    ↓
[腾出空间] → [继续新任务]
```

压缩策略：
- 保留决策结论（搜索发现了什么）
- 丢弃中间过程（怎么搜的、搜了什么无效路径）
- 摘要压缩比可达 80%+

## 4. 与其他 Token 优化机制的关系

| 机制 | 作用 | 触发时机 |
|------|------|---------|
| **Prompt Cache** | 静态背景复用，成本降低 80%+ | 首次加载后持续有效 |
| **Context Auto-compaction** | 动态压缩超限上下文 | Token 逼近阈值 |
| **Sub-agent 隔离** | 中间日志自行消化，不回传主进程 | 大范围任务开始时 |
| **金鱼记忆** | 每次会话重置，无持久上下文 | 每次新会话 |

## 5. 工程实现要点

- **阈值设定**：需要根据模型 Context Window 大小和任务类型动态调整
- **压缩质量**：摘要必须保留决策所需的关键信息，不能丢失核心结论
- **与模型能力联动**：依赖模型的摘要/推理能力，压缩质量上限由模型决定
