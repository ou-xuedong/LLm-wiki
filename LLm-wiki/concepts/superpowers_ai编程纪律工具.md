# 🧠 Superpowers（AI 编程纪律工具）

## 1. 定义与核心原理
Superpowers 是一套给 AI 植入"资深工程师内化纪律"的机制。核心问题：AI 很擅长"完成你说的"，但不擅长"做你没说的但应该做的"。Superpowers 通过强制机制弥补这一缺陷。

**本质**：让 AI 从"被动执行"变成"主动严谨"。

## 2. 三大核心机制

### 机制一：强制 TDD（测试驱动开发）
- **逻辑**：先写测试，再写实现。测试写不出来 = 对功能理解不清楚
- **强制方式**：不写测试就删代码（听起来暴力，但有效）
- **核心价值**：逼你在动手之前把功能想清楚

### 机制二：标准化工作流
完整功能开发必须经过五个阶段：
1. **头脑风暴**：把所有可能情况列出来
2. **计划**：确定实现路径
3. **执行**：按计划写代码
4. **审查**：自我 review，找问题
5. **收尾**：整理、归档、更新文档

### 机制三：可复用能力模块
- 将稳定的 API 请求封装、日志格式、错误处理等固定为可复用模块
- 避免每次让 AI 重新生成（每次生成都可能略有不同 = 代码风格割裂根源）
- 沉淀越多，AI 产出质量越高

## 3. 与主人方向的关联
主人做**亚马逊投流决策助手**，Superpowers 逻辑可映射：

| Superpowers 机制 | 投流助手映射 |
|---|---|
| 强制 TDD | 策略上线前必须先有"回测"（历史数据验证） |
| 标准化工作流 | 投流策略开发：竞品分析 → 关键词规划 → 出价策略 → 复盘归档 |
| 可复用能力模块 | 投流策略模板、关键词词库、出价策略库 |

**核心启示**：AI 投流工具"不主动思考边界情况"是最大痛点，需要强制机制弥补。
