# 🧠 智能体记忆架构（Agentic Memory Architecture）

## 1. 定义与核心原理

智能体记忆架构是指 AI Agent 在长时间自主运行过程中，**如何获取、存储、检索和遗忘信息**的系统性设计。

核心矛盾：**模型的"生命"是单次推理，但任务的时域可能是数小时、数天甚至永远在线。**

记忆架构的本质是**在模型外部构建持久化状态层**，让模型像人一样既有短期工作记忆、又有长期知识沉淀。

---

## 2. 主流分层记忆模型

参考人类记忆研究，业界主流分为三层：

### 2.1 工作记忆（Working Memory）

| 属性 | 说明 |
|---|---|
| 对应人类 | 前额叶短时记忆 |
| 存储位置 | 模型内部隐状态（Hidden State）|
| 容量 | 受上下文窗口限制（当前最强 ~100万token）|
| 生命周期 | 单次推理内，推理结束即消失 |
| 典型实现 | StreamingLLM 的"注意力锚点"机制 |

### 2.2 情景记忆（Episodic Memory）

| 属性 | 说明 |
|---|---|
| 对应人类 | 海马体记忆 |
| 存储位置 | 外部数据库（向量库/关系库）|
| 存储内容 | 具体事件、用户交互历史、任务执行记录 |
| 检索方式 | 向量相似度召回 |
| 生命周期 | 可长期保留，按需召回 |

### 2.3 语义记忆（Semantic Memory）

| 属性 | 说明 |
|---|---|
| 对应人类 | 大脑皮层长期知识 |
| 存储位置 | 外部结构化存储（JSON/YAML/知识图谱）|
| 存储内容 | 规则、偏好、身份信息、宪法/原则 |
| 检索方式 | 精确查询（非向量）|
| 生命周期 | 永久或半永久，几乎不遗忘 |

## 3. 关键技术方案

### 3.1 RAG + 向量召回（最成熟）

```
用户输入 → 编码为向量 → 在向量库（Qdrant/Milvus）中检索相似文本 → 注入上下文
```

适合：外部知识补充、文档问答、跨会话上下文延续

### 3.2 模型自主读写（Anthropic MCP方向）

```
模型推理 → 判断需要记忆 → 调用工具写入外部存储 → 下次推理时读取
```

适合：Agent 长期运行场景，模型自己决定什么时候记住

### 3.3 递归摘要压缩

```
历史对话 → 定期调用LLM压缩摘要 → 摘要替代原始对话存储 → 节省token
```

适合：超长对话，成本敏感场景

### 3.4 记忆层次自动升降级

```
新事件 → Working Memory → 重要事件升级到 Episodic → 核心规则沉淀到 Semantic
不重要内容 → 自然遗忘或主动清理
```

## 4. 行业应用场景

| 场景 | 记忆需求 |
|---|---|
| Claude Code 编程助手 | 跨会话记住项目结构、用户偏好、编码规范 |
| 投流助手质量引擎 | 宪法（Semantic）+ 验证结果（Episodic）+ 当前任务（Working） |
| AI Copilot 全天候助手 | 用户长期偏好、任务历史、项目上下文 |
| 自动驾驶/具身智能 | 传感器历史、地图更新、规则约束 |

## 5. 代表性项目

- **MemGPT**（UC Berkeley，2023）— 首个将 OS 虚拟内存概念引入 LLM 记忆管理
- **ChatDB**（2023）— 用 SQL 作为外部记忆存储，实现显式关系推理
- **Anthropic MCP**（Model Context Protocol）— 模型调用外部工具读写记忆的行业标准协议
- **Claude Code AutoDream**（2026-03）— 自动管理跨会话记忆文件，防止记忆膨胀

---

## 6. 争议与开放问题

- **记忆的真实性**：向量召回的内容是否准确？如何防止记忆"污染"？
- **遗忘机制**：什么信息该遗忘？手动 vs 自动 vs 模型自主决定？
- **记忆安全与隐私**：长期记忆中的敏感信息如何保护？
- **记忆与身份**：如果 Agent 的记忆被复制或篡改，"它"还是"它"吗？
