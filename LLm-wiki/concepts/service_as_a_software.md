# 🧠 Service-as-a-Software

## 1. 定义与核心原理

**Service-as-a-Software（SaaSware）** 是 AI 时代的核心商业模式升级，由 [[AI创业十三条军规-核心评估准则|AI创业十三条军规_核心评估准则]] 第 5 条提出。

**核心定义**：
> 对外提供极重度的"服务外包（BPO）"体验，按"任务完成数"或"结果价值"向客户收费；对内用 AI Agent 替代人力执行，享受纯软件的无限扩展与零边际成本。

**与传统 SaaS 的本质区别**：

| 维度 | 传统 SaaS | Service-as-a-Software |
|---|---|---|
| 交付物 | 工具 / 席位 | 结果 / 任务完成 |
| 定价单位 | 月费 / 账号 | 按任务 / 按结果价值 |
| 客户感知 | "我买了一个软件" | "我买了一个服务，幕后是软件" |
| 边际成本 | 零 | 零（但对外看起来像服务） |
| LTV 逻辑 | 续费率 | 交付量 × 任务价值 |
| 对标 | Salesforce / HubSpot | 咨询公司 + BPO + AI Infra 混合体 |

## 2. 关键技术路线与变体

### 路线 A：AI 替代人力服务
- 前端是服务合同（像请律师/咨询师），后台是 AI Agent 执行
- 代表：[[harvey_垂直法律agent|harvey_垂直法律agent]]（AI 替代 junior paralegal）

### 路线 B：按任务完成数计费
- 每完成一个指定任务收一笔费用
- 代表：Adept、RPA + LLM 的流程自动化产品

### 路线 C：按结果价值抽成
- 以客户获得的增量价值为计费基数
- 代表：广告代理（媒体费抽成）、电商 CPS、[[投流决策助手_跨境电商ai决策|投流决策助手_跨境电商ai决策]] 的 L2 结果分润

### 路线 D：混合分层模型（推荐）
- 基础席位费 + 按结果抽成 + 深度合作股权
- 代表：本方向的 L1+L2+L3 三层定价
- 优势：平衡现金流稳定性与结果导向

## 4. 代表性论文/项目

- **Sam Altman 公开观点（2024）**：AI 时代的软件公司将更像 BPO + SaaS 的混合体
- **a16z / Sequoia 投资主题（2026）**：顶级 VC 明确追捧"按结果计费"的 AI 公司
- **AI 创业十三条军规第 5 条**：[[AI创业十三条军规-核心评估准则|AI创业十三条军规_核心评估准则]]
- **Harvey 商业模式拆解**：按合同审查件数 + 席位混合计费
- **技术底座开源参考**：CrewAI、AutoGen 等多 Agent 框架

## 5. 争议与开放问题

- **归因难题**：按结果计费需要可量化、可归因的增量价值基线，技术实现门槛高
- **客户信任**：客户如何验证 AI Agent 真的完成了任务而非幻觉
- **法律责任**：AI Agent 做错了谁来担责（与 [[agent协商层|agent协商层]] 共同的监管盲区）
- **定价博弈**：结果分润的比例给客户多少、给自己多少（行业无共识）
- **与传统 BPO/咨询冲突**：SaaSware 本质在替代传统服务业，会遇到既得利益阻力
- **毛利波动**：AI 模型成本高峰时，按结果计费会挤压毛利
- **估值模型模糊**：VC 在"SaaS 估值倍数"与"服务公司估值倍数"之间如何拿捏
