# 🧠 亚马逊VC全托管商业模式

## 1. 定义与核心定位

**VC（Vendor Central）** 是亚马逊邀请制的B2B供应商平台，主要面向大品牌和制造商，本质是"全托管出海服务"。与FBA（Fulfilled by Amazon）相比，VC模式下供应商是"向亚马逊供货"，亚马逊是"买家"。

**核心价值主张**：让优质供应商"拎包入住"，专注产品创新，其余全部外包。

## 2. 商业模式架构

### 三大参与方
```
供应商（30%利润） ←—— 利润分成 ——→  VC方（70%利润，含兜底服务）
                ↓
         亚马逊VC平台
         （采购方）
```

### VC方提供的全托管服务
1. 头程（发往亚马逊仓）
2. 海外仓
3. VC账号运营
4. Woot联秒
5. 广告投放
6. 运营优化
7. **60天兜底垫付**（无论盈亏，优先垫付货款给供应商）

### 供应商只需提供
- 优质创新产品
- 其余全不用管

## 3. VC爆款模型（30天双子体连秒）

**操作流程**：
| 阶段 | 时间 | 操作 |
|------|------|------|
| 建结构 | 1-7天 | 1个父体 + 2个子体（均不发货FBA） |
| 跟卖布局 | 8-15天 | VC跟卖子体1 + Woot跟卖小号 + 小号跟卖子体2 |
| 补货到仓 | 16-23天 | 货到VC仓和小号FBA仓（Woot划走FBA库存） |
| 双秒杀 | 24-31天 | 同时开启VC + Woot双秒杀 |

**与FBA的核心区别**：
- 无FBA仓储费
- T+7快速回款（vs FBA 14-21天）
- BD连秒可从7天起步（最长28天）
- VC采购价通常低于FBA零售价（亚马逊赚差价）

## 4. 与投流决策助手的协同关系

> VC全托管 = 供给侧基础设施（选品+供应链+仓配）  
> Listing智能体 = 内容侧基础设施（关键词+合规+转化）  
> **投流决策助手 = 需求侧基础设施（广告+决策+增长）**

三者构成亚马逊卖方铁三角，VC全托管供应商是投流决策助手的潜在客户群。

## 5. 核心方法论（754310体系）

**7个维度**：做人 / 做生意 / 做电商 / 做亚马逊 / 做产品 / 做大盘 / 做词

**7个页面**：主图 / 标题 / 详情 / 榜单 / 后台 / 广告 / 邮件

**5个问题**：钱在哪 / 被谁赚 / 怎么抢 / 用什么工具 / 谁执行

**4个环节**：市场调研 / 视觉 / Listing / 广告

**3重节奏**：物流节奏 / 上新节奏 / 灌流量节奏

**1门广告学问**：
- 大词+精准词+长尾词三层覆盖
- 节流+截流+劫流三档操作
- SP+SB+SD+AMC四广告全开

**0之无极**：持续学习能力 + 底层平台思维
