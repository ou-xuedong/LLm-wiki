# 从 SFT 到继续预训练：大模型领域知识注入的原理与实战

> 作者：李剑锋（机智流）
> 发布：2026年5月14日

## 核心观点

### SFT vs 继续预训练

- **SFT 核心作用**：行为模式调整（"怎样回答"），而非注入大量新知识
- **继续预训练目标**：让模型在参数层面真正熟悉垂类术语、表达方式和知识分布（"更懂什么"）

### 何时值得做继续预训练

1. 模型对领域术语明显不熟悉
2. 现有方法（提示词/RAG/Agent）难以弥补
3. 手中有大量高质量、非公开的原始领域数据

### GLM-5 训练流程启示

- 基座训练 = 通用语料（18T）+ 代码推理（9T）+ Mid-training + Sparse Attention Adaptation
- Post-training = SFT + 强化学习（推理/Agent/工具调用）

### 医疗案例 MEDITRON-70B

继续预训练语料设计原则：
- 领域性强：论文 + 临床指南（后者更贴近决策实践）
- 加入少量通用域回放数据防止灾难性遗忘（catastrophic forgetting）
- 最终数据混合需通过实验验证，非越多越好

关键结论：继续预训练解决"让模型更懂医疗"，task-specific SFT 解决"让模型更会完成具体医疗推理任务"。两者分别作用于"领域适应"和"任务适应"不同层面。

### 实战要点

- 训练数据格式：jsonl，每行 `{"text": "..."}`
- 分词时手动添加 eos_token 标注文本边界
- 固定长度分块（block_size），因果 LM 的 labels = input_ids 拷贝
- 全量训练学习率偏保守（1e-5），QLORA 学习率通常高一个量级（1e-4）
- QLORA 核心价值：大幅降低显存占用，在单卡场景更易落地
