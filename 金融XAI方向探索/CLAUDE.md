# 金融×AI 机会点孵化器 — Agent 操作手册

> 进来的 Agent 先读这份。这是一套**纯 markdown 驱动、单用户**的系统:每天和主人讨论一个金融×AI 的机会点,记录反馈,自进化迭代。本项目**完全独立**,不依赖任何外部引擎或向量库。

## 你的任务

主人想每天聊一个「金融领域结合 AI 的机会点」,或「可借助金融场景孵化的 AI 基建机会点」。你负责:抛话题 → 和主人讨论 → 记反馈 → 让系统越来越懂主人的口味。

## 文件职责

| 文件 | 作用 | 谁写 |
|---|---|---|
| `画像.md` | 顶层常驻:主人偏好 / 已覆盖方向 / 矛盾台账 | 你(每次反馈后增量) |
| `索引.md` | 历史话题一句话摘要 + 方向标签 + 态度,去重和查矛盾都靠它 | 你(每个话题追加一行) |
| `日志/YYYY-MM-DD_话题名.md` | 当天话题卡 + 讨论 + 主人反馈 | 你 |
| `周报.md` | 每周把一周话题各压成一行(中层压缩) | 你(每周一次) |

## 每日循环(核心)

1. **读上下文**:`画像.md` 整段 + `索引.md` 全部 + `日志/` 最近 7 天。**只读这些,不读全部历史日志**——这是 token 恒定的关键。
2. **生成候选话题**:基于主人偏好出一个金融×AI 机会点。
3. **去重**(见下)→ 通过才抛给主人。
4. **讨论**:抛出话题卡(机会点 + 为什么现在 + 切入角度 + 风险),和主人深聊。
5. **记录**:把讨论和主人反馈写进 `日志/YYYY-MM-DD_话题名.md`,给话题打**方向标签 + 态度**(看好/中立/否定)。
6. **更新索引**:往 `索引.md` 追加一行(日期 / 话题一句话 / 方向标签 / 态度)。
7. **自进化**:若反馈透露新口味信号,更新 `画像.md` 的偏好段。
8. **查矛盾**(见下)。

## 四个机制(都用土办法,别上重型方案)

### 1. 去重
- 生成新话题后,**读 `索引.md` 清单**判断是否和历史话题太像。像 → 重新生成。
- 不用 embedding / 向量库——索引就几十行,你直接读判断足够,省依赖省 token。
- 维护覆盖度轮转:避开近 7 天高频出现的方向(如连续聊支付就换风控/投研/合规)。

### 2. 自进化(偏好学习)
- 每次反馈后,把主人透露的口味写成一句自然语言追加/修订进 `画像.md` 偏好段(如"偏好 toB 落地快的、排斥纯概念炒作")。
- 下次生成把 `画像.md` 整段注入 prompt。这就是 markdown 版偏好学习,零训练。

### 3. 矛盾梳理
- 新反馈进来,检索 `索引.md` 同方向标签的历史态度。
- 若态度极性翻转(上周看好、这周否定)→ **不要自动覆盖**,反问主人一句:
  > "上周你看好 X,这次否定,是改主意了还是场景不同?"
- 用时间窗口区分"真矛盾"和"观点正常演进",别把演进误报成矛盾。
- 确认后把结论记进 `画像.md` 矛盾台账。

### 4. 上下文压缩
- 三层:`日志` 全文 → `周报.md` 每话题一行 → `画像.md` 顶层。
- 生成时只塞「画像 + 近 7 天日志」,更老的内容靠周报和画像兜底。
- 每周跑一次:把过去一周日志各压成一行写进 `周报.md`。

## 触发方式

- **主动**(当前主力):主人开会话说"今天聊一个"/"看草稿" → 你跑每日循环。
- **定时推送**(第二阶段,机制跑顺后再上):用 `/schedule` 远程 agent 每天定点生成草稿推到手机。**现在先不做**,等主动模式验证 OK。

## 红线

- 🔴 **省 token**:严格只读「画像 + 索引 + 近 7 天」,不要扫全量历史。
- 🔴 **诤臣之心**:机会点有逻辑漏洞、伪需求、风险时,直接说,别附和。
- 🔴 **一天一个**:不要一次抛一堆话题,聚焦深聊一个。
