用 Slack 跑通你的第一个多 Agent 协作任务

先看 Anthropic 内部真实玩法，再用最傻瓜的 Webhook 方式，让本地 Claude Code「队长」派出 3 个子 Agent，把成果一条条汇进你手机的 Slack 频道。全程约 15 分钟。

1 Anthropic 内部怎么用 Slack 协作

一句话：把 Slack 频道当成「人和一群 Agent 共用的工作台」。

① Slack = 共享上下文的协作面
人和多个 Agent 泡在同一个频道/线程里，看的是同一份上下文。@一下 Claude Code ，它就能就地读取这条讨论的来龙去脉接着干——不用你复制粘贴搬运信息。

② 官方刚把这条路产品化：Claude Code in Slack
Anthropic 推出了 研究预览 ——在 Slack 里 @Claude，就能就地拉起一个 Claude Code 会话，把频道里的 bug 报告、需求讨论直接喂进去开干，多人还能在同一线程里围观、接力。

③ 多 Agent 的核心范式：一个队长 + 一群分身
官方 Agent Teams （2026-02 随 Opus 4.6 推出）让一个会话当「队长」，派出多个 子 Agent 并行干活、共享一张任务清单、彼此通气。Slack 在这套里扮演的，就是大家把进度和成果同步出来的「公告板」。

⚠️ 真相提醒：官方的「Claude Code in Slack」是研究预览、可能要排队或需工作区管理员安装，不一定对你开放。所以下面我们走你今天就能完全自己跑通的等价路线。

2 你要跑的「第一个」长什么样

本地 Claude Code 当队长，派 3 个子 Agent，成果通过 Webhook 汇进 Slack 频道，你在手机上围观。

🧠 队长 Agent（你的 Claude Code）
↓ 拆任务 · 并行派活
子 Agent 1方向 A 调研
子 Agent 2方向 B 调研
子 Agent 3方向 C 调研
↓ 各自把要点 POST 出来
💬 Slack 频道 #ai-agents
📱 你在手机 Slack 里，看 3 条成果 + 1 条队长汇总，陆续冒出来

把成果送进 Slack 用的是 Incoming Webhook ——Slack 给你一个专属网址，往它发消息就出现在频道里。只发不读，所以最简单、零审批，最适合跑通「第一个」。两周后想升级成 Agent 能读也能写的双向协作，再换 MCP 方案（见文末「下一步」）。

3 傻瓜式 5 步跑通

照着点、照着粘贴即可。文件都已放在本文件夹里。

1

建一个 Slack App + 频道（约 3 分钟）

  • 先在 Slack 里建一个频道，名字叫 #ai-agents（左侧「频道」旁 ➕ → 创建）。
  • 电脑浏览器打开 api.slack.com/apps → Create New App → From scratch。
  • 名字随便填（如 AgentBus），选中你自己的工作区 → Create。
2

打开 Incoming Webhook，拿到专属网址

  • 左侧菜单 → Incoming Webhooks → 把开关 Activate 打开。
  • 页面底部 Add New Webhook to Workspace → 选频道 #ai-agents → Allow。
  • 复制生成的网址，形如 https://hooks.slack.com/services/T000/B000/xxxx。
⚠️ 这串网址 = 谁拿到谁能往你频道发消息，当密码保管，别提交到 git、别发群里。
3

把网址存好 + 发一条测试（约 1 分钟）

在本文件夹里打开终端，把网址存进一个文件（脚本会自动读它）：

# 把下面的 https://hooks.slack.com/... 换成你刚复制的网址
echo "https://hooks.slack.com/services/T000/B000/xxxx" > .slack_webhook

# 发一条 hello 验证通道（看手机 Slack 是否冒出消息）
./slack_post.sh "✅ 通道打通：Agent 总线已上线"
📱 手机 Slack 的 #ai-agents 里看到这条消息 = 通道成功。看不到就回查第 2 步选的频道是否一致。
4

把这段指令整段复制给 Claude Code

在本文件夹里启动 claude，把下面整段粘进去回车。它会让队长派 3 个子 Agent，各自把成果发进 Slack：

你现在是"队长 Agent"。目标：为我的"前沿技术研究"
文件夹提 3 个值得做的新选题方向。请这样做：

1. 用 Task 工具并行派出 3 个子 Agent，各自独立
   提 1 个方向，每个给出：方向名 / 为什么值得研究
   / 一句话切入点（≤80 字）。

2. 每个子 Agent 结果一回来，立刻用 Bash 执行（在本
   文件夹下）：
   ./slack_post.sh "子Agent-N · 方向名" "该方向的要点"

3. 三个都发完后，你作为队长再发一条总结：
   ./slack_post.sh "队长 · 汇总" "三个方向各一句 + 你最推荐哪个及理由"

4. 全程让我在 Slack #ai-agents 里看到这 4 条消息陆续出现。
   每发一条，在终端也告诉我发了什么。

↑ 这段指令也存成了同目录的 demo指令.txt，手机上也能复制。

5

掏出手机，看多 Agent 成果落地

打开手机 Slack 的 #ai-agents，你会看到 3 条子 Agent 成果 + 1 条队长汇总 陆续冒出来——这就是一次完整的「多 Agent 经由 Slack 协作」。

🎯 跑通的标志：频道里出现 4 条带不同 *标题* 的消息，且队长那条对 3 个方向做了取舍推荐。

4 跑通之后，怎么升级

从"只发不读"进阶到"能读能写"的真双向协作。

  • 升级一·双向协作（贴近 Anthropic 范式）：把 Webhook 换成 Slack MCP server ，Agent 就能读你在频道里的追问、并回帖。届时你在手机上 @它一句"方向 B 再深挖"，子 Agent 能接着干。
  • 升级二·定时自动：把这段队长指令挂成定时任务（如每早 9 点），每天自动给频道推一批新选题。
  • 升级三·真·Agent Teams：在支持的版本里改用官方 Agent Teams，让子 Agent 之间也能直接通气、共享任务清单，而不只是各自汇报。

想做哪个升级，跟我说一声，我再给你出对应的傻瓜步骤。

本文件夹配套文件
