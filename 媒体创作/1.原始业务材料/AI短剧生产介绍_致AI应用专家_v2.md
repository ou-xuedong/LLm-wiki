2.1 整个微短剧行业 · 千亿规模
根据广电总局 2026 年 2 月数据:2025 年微短剧市场规模突破千亿,国内用户近 7 亿人。这个规模已经超过了中

微短剧的特征:
单集 1-3 分钟,每部 60-100 集
竖屏 9:16,移动端为主
题材以"爽文"为主:逆袭、复仇、霸总、重生、玄幻、女频虐恋
商业模式:用户付费(IAP)+ 广告分账(IAA)+ 平台分账
2.2 AI 短剧 · 在千亿短剧里"分裂"出来的新物种
在传统真人短剧之外,2025-2026 年AI 制作的短剧异军突起,分两大类:
① AI 真人短剧
形态:用 Seedance 2.0、可灵、即梦等模型生成看起来像真人拍的画面,但实际全是 AI 生成的
转折点:2026 年 2 月,字节跳动 Seed 团队发布 Seedance 2.0,让 AI 视频从"玩具级片段"跃升为"工业级成
片",此后两个月行业剧烈变动
典型成本:精品仿真人剧 1500-3000 元/分钟,走量剧更低
头部代表作:《斩仙台 AI 真人版》(6 天破亿)、《兴安岭诡事》(首部付费 AI 短剧,5000 万播放)、《奶团太
后宫心计》(2.2 亿播放)
② AI 漫剧
形态:动漫/二次元画风,涵盖动态漫、表情包漫、2D/3D 动画短剧
规模:2025 年市场规模 189.8 亿元,同比 +276.3%。2026 年预计突破 200 亿
产能:2026 年 1 月,国内 AI 漫剧单月上线 14634 部,日均超 470 部。3 月仅红果一个平台漫剧日均上新
2000 部,是真人剧的 20 倍

## 第 9 页

3.2 三个最关键的概念(必须搞懂)
🎬 抽卡(本文档反复出现)
🎬 业务说: 同一组提示词喂给 AI 视频模型,每次出来的结果都不一样,有时好,有时崩,要不断"抽"才能抽
到能用的版本。
💻 技术翻译: 非确定性生成式 API 调用。Temperature>0,无法保证幂等。一次失败的根因不可观察,
只能重试。
抽卡是 AI 短剧最大的成本中心,也是最大的不确定性来源。一个 3 秒镜头平均要抽 1.5-3 次才能抽到能用的。
🎬 分镜(决定抽卡成败的源头)
🎬 业务说: 把整集剧本拆成一个个 2-8 秒的小镜头,每镜头标注景别、运镜、动作、台词。分镜写得好,
抽卡就顺利;分镜写得糙,抽卡就疯狂崩。
💻 技术翻译: 把高级需求拆解为一系列原子任务。每个任务有标准化的输入参数(prompt、首帧、运
镜)。任务粒度越细、约束越明确,API 成功率越高。这是后续"抽卡前准备做到极致"方法论的核心。
🎬 角色一致性(最大技术难题)
🎬 业务说: 同一个男主,在第 1 个镜头长的是这张脸,第 5 个镜头变成另一张脸,观众一看就出戏,整集废
掉。
💻 技术翻译: 跨多次 API 调用之间维持一个"实体"在视觉上的恒定状态。每次抽卡是独立请求,无法天
然继承上一次的"角色定义",必须靠参考图、LoRA、主体库等技术手段强制锁定。

## 第 20 页

07 核心痛点 · 翻译成程序员能理解的工程问题
读这一章的目的:把会议纪要里的"幻觉""不可复制""猜谜游戏"翻译成你熟悉的工程问题。每个痛点都给出业
务侧描述 + 技术翻译 + 我们的初步假设。 
7.1 痛点 A:非确定性生成,无可观察性
🎬 业务说: 同样的提示词、同样的首帧图,在 Seedance 2.0 上跑两次,结果差异很大。失败的时候我们
不知道为什么失败,只能重试。
💻 技术翻译: 生成式 API 的 stochastic 特性 + 黑盒模型,导致缺乏可观察性。Failure mode 不可枚
举、不可定位。这种 API 在工程上属于"有损调用",必须配套重试 + 选优 + 监控机制。
我们的初步假设
无法消除非确定性,但可以把它压缩到尽可能小的范围
方法:在抽卡前把所有可控因素(角色、场景、构图、首帧、提示词)固化到极致,只把"运动"这一个变量留
给 Seedance
同时,需要建立失败模式分类(脸崩、肢体崩、动作变形...),作为 Prompt 优化的反馈信号
7.2 痛点 B:成功经验不可复制(无样本沉淀机制)
🎬 业务说: 组员小王今天抽到一个特别好的镜头,用了某个绝妙的提示词,但他自己也说不清为什么这次
就行了。第二天小李遇到类似镜头,只能从零摸索。
💻 技术翻译: 缺乏experience replay机制。每个 API 调用是独立事件,成功路径没有被结构化记录,无
法形成数据资产。这等于每个员工都在本地训练自己的"私有模型",组织没有积累。
我们的初步假设
需要一个"成功样本库":每个抽到的好镜头,自动记录提示词、参数、首帧、抽卡次数、负责人
新镜头要抽时,先在样本库里找相似镜头的成功 Prompt 作为参考
长期可以做"Prompt 模式提取":从大量成功样本里反向归纳出结构化的"Prompt 模板"
