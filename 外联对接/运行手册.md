# 外联对接 · 运行手册

本目录文件：
- `对外说明_CN.md` / `对外说明_EN.md` — 对外一页纸（产品定义变了就同步更新）
- `VC对接追踪.md` — CRM 追踪表，去重 + 状态流转的唯一依据
- `mailer.py` — Gmail 收发引擎
- `.mail_config` — 凭证（gitignore，本地填，绝不提交）

---

## 流程 A · 每日找新 VC + 草拟（token 受控）

每天对小昭说「**跑一下 A 流程**」即可，步骤固定：

1. 读 `VC对接追踪.md`，列出已接触过的 VC（去重）。
2. **只搜 1 家**没接触过的新 VC：拿到官网 + 1 个社媒，找到对接/投递邮箱。
   - 🔴 token 闸：最多查官网 + 1 社媒，不深爬、不连环搜。
3. 把这家写入追踪表，状态 `待发`，填来源链接。
4. 基于 `对外说明_CN/EN.md` 草拟一封邮件（中或英按对象定），呈给主人。
5. 主人确认后，小昭执行：
   ```
   python3 mailer.py send --to <邮箱> --subject "<主题>" --body-file <草稿文件>
   ```
6. 发送成功 → 追踪表状态改 `已发`，更新「最后更新」。

## 流程 B · 收信 + 草拟回复

每天对小昭说「**跑一下 B 流程**」：

1. 拉回信（BODY.PEEK，不标已读；该邮箱旧广告多，**必须收窄**）：
   ```
   # 只看追踪表里已发 VC 的回信（最省 token，推荐）
   python3 mailer.py fetch --since-days 14 --from "vc1@x.com,vc2@y.com"
   # 或泛拉最近几天（不带 --from）
   python3 mailer.py fetch --since-days 3 --limit 20
   ```
2. 小昭一句话告诉主人每封 VC 回信说了啥。
3. 匹配到追踪表对应行 → 状态改 `已回`。
4. 草拟回复，呈给主人。
5. 主人确认 → 同流程 A 第 5 步发送 → 状态改 `跟进中`。

---

## 凭证安装（一次性）

1. Gmail 开两步验证 → `myaccount.google.com/apppasswords` 生成 16 位应用密码。
2. 复制 `.mail_config.example` 为 `.mail_config`，填入 EMAIL 和去空格的 16 位密码。

## 关于「每天自动」

发送/回复都有主人确认环节，做不到全程无人值守。
可用 `/schedule` 每天定时触发到「草拟待确认」，主人确认后才真发。
